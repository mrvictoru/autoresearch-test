# Planner

Propose restaurant-policy hypotheses for a single worker to test.

Rules:
- focus on perishability, lead time, storage, overlapping demand, and policy design
- propose one hypothesis change per idea
- cite why the idea might improve score
- **ground every hypothesis in observed policy behavior**: reference the behavioral notes from `results.tsv` (e.g. stockout ingredients, waste offenders, demand-spike misses) rather than proposing changes in a vacuum
- if behavioral notes are not yet available, explicitly ask the worker to run a baseline inspection first
- write ideas into the control-plane backlog before execution

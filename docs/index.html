Overview
Python conference talk

A Python pattern for bounded, reproducible experimentation.

This 15-minute talk uses mrvictoru/autoresearch-test to show how Python turns AI-assisted edits into a measurable loop with explicit boundaries.

Core architecture in this repo

1.tasks.py defines an immutable benchmark with perishability, lead times, and shared storage limits.
2.restaurant_train.py is the mutable surface and must expose build_policy().
3.restaurant_eval.py loads and evaluates the policy with stable metric output.
4.Keep/discard is benchmark-driven via score comparisons and a frontier ledger.
The Python value proposition: reproducibility first, autonomy second.

Why it matters

  • Turns prompt iteration into a repeatable software experiment.
  • Makes benchmark assumptions and mutation boundaries explicit.
  • Keeps outcomes auditable through stable metrics and run logs.

Proposal framing: use Python to encode the loop, not just generate edits.
Method
Ratchet loop
Method

Hypothesis → run evaluator → parse metrics → decide.

1. Propose

Change only the policy module and keep one clear hypothesis per run.

2. Run

Execute the same evaluator entrypoint against the mutable experiment file.

3. Score

Read score and METRIC_JSON from a stable results block.

4. Ratchet

Keep only strict improvements; discard or revert everything else.

Benchmark / evaluator / policy split

  • tasks.py: immutable simulator and scoring context
  • restaurant_eval.py: immutable loader + metric printer
  • restaurant_train.py: mutable policy via build_policy()

Why this is reproducible

  • fixed seeds and deterministic scenario generation
  • bounded mutation surface
  • stable output contract for automation
Bounded experimentation is the key engineering pattern.

Mutable policy file with build_policy() returning the active strategy.

Operational contract (from repo docs)

  • Read program.md first.
  • Edit only restaurant_train.py.
  • Use evaluator output as source of truth for score.
docker compose run --rm autoresearch python -m autoresearch.experiments.restaurant_eval \ --experiment autoresearch/experiments/restaurant_train.py --- RESULTS --- score -28735.040000 service_level 0.390830 METRIC_JSON: {"score": -28735.04, "service_level": 0.39082969432314413, ...}

Same benchmark + same evaluator output contract = comparable runs.

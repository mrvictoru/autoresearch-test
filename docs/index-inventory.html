Operations Pain Points餐飲營運痛點

1.Information fragmentation: orders, supplier constraints, lead times, and decay rules live separately.資訊碎片孤島：訂單數據、供應商約束、交期與食材物理（shelf-life）特性散落各處。
2.Double-sided margin leakage: over-ordering causes immediate spoilage, under-ordering causes high peak-hour stockouts.雙向利潤流失：補貨多則過期浪費、補貨少則尖峰時段缺料，皆嚴重傷害門店營收。
3.Multi-variable constraint planning: human operators struggle to manual-balance capacity limits and spoilage.多約束人工計算：店經理極難靠人工手動精準算清「物理倉儲容積上限」與「易腐食材到期日」。

Four Layers of Agent DefensesAI 真實研發沙盒：四重安全性防線

1. Non-Tampering Git Guard1. 接口防篡改防線 (`git diff`)
Verify inputs and block changes to test suites or eval modules using git diff --name-only. 在運行前強制進行變動審計，確保 AI 只修改 `restaurant_train.py`。若試圖篡改測試代碼或評估器，即刻中斷判定舞弊。
2. Subprocess Isolation2. 進程級獨立沙盒 (`subprocess`)
Spawn a clean subprocess per trial to clear Python import cache and prevent dirty global variables. 利用子進程隔離加載候選策略模組，防止 Python `sys.modules` 導入緩存和全域變量殘留，保證環境 100% 潔淨。
3. Pinned Pseudo-Seeds3. 隨機數種子釘扎 (Deterministic Seeds)
A Gaussian demand stream pinned with deterministic random seeds to block AI from cheating on lucky draws. 在 `tasks.py` 釘扎客流隨機數種子。每次評估都在完全相同的客流噪聲與供應交期中考驗，消除巧合干擾。
4. Atomic Rollback Guard4. 物理級原子回滾 (`git reset`)
Record results. If score decreases, invoke git reset --hard HEAD~1. Keep and commit only when a new high record is made. 在 `control_plane.py` 對接 Git：每次測試前存檔。若評估分數未超越 `results.tsv` 的歷史最高分，一秒回滾！唯有創新高才允許提交。

Multi-Factor Constraints多因庫存物理約束

  • Stochastic Demand Peaks: weekly cyclical consumer demand spikes with varying weekday consumption profiles.隨機需求起伏（Stochastic Demand）： 波動性客戶流量，週一至週四低平、週末暴烈上升。
  • Physical Volume Thresholds: shared storage constraints. Overflowing triggers massive holding cost penalties.共享倉庫容積上限（Capacity Barrier）： 倉儲空間物理極限固定。一旦囤貨溢出，觸發溢出保管罰金。
  • Perishable Spoilage Window: strict food perishability decay curves. Excess supply degrades directly into zero-value waste.食材到期耗損（Spoilage Penalty）： 核心食材（生菜、起司、洋蔥、麵體）保鮮期極短，過量補貨即刻報廢。
  • Global Objective Optimization: maximize cashflow and customer orders filled while curbing physical decay costs. 全域財務優化函數（Objective Function）：
    $$\text{Net Profit} = \sum (\text{Revenue}) - \text{Holding Cost} - \text{Spoilage Penalty} - \text{Stockout Opp Loss}$$

Best Practices for AI R&D1. AI 自動研發工程實踐啟示

  • Isolate State & Runtime: Process-level isolation (subprocess/dynamic import) is the only baseline to prevent code mutation polluting memories.消除進程與狀態污染：必須實施子進程隔離載入及依賴熱插拔，拒絕全局變量污染與 sys.modules 快取中毒。
  • Erect Security Boundaries: Without git reset rollback and file non-tampering audits, agents will easily bypass metrics or cheat test suites.構築物理級防護邊界：必須部署 Git 原子備份與篡改阻斷（git diff），防止大模型篡改測試集代碼來自動舞弊。
  • Pin Deterministic Seeds: Lock randomized demands using pinned pseudo-seeds. Fluctuating random factors will completely mislead optimization.堅守確定性種子（Seed Pinning）：隨機噪聲是 AI 優化的大敵。必須釘死客流種子，否則策略微調方向將完全失焦。

Production Scalability2. 連鎖門市橫向拓展物理防線

  • Replicability: Fully-dockerized templates allow one-click deployments to thousands of multi-city restaurant networks.容器化極簡擴展：純 Docker 容器化 + Git 沙盒模式，無痛對接多地區或連鎖門店。
  • Protocol Compatibility: Standardized JSON schemas align perfectly with external POS, supplier logistics, and warehouse ERP interfaces.標準接口兼容：嚴格 JSON schema 輸出，無縫對接 ERP 系統與供應商採購通道。
  • Dynamic Swapping: Run hot-swapped Python modular policies under production, achieving zero-reload zero-stop updates.動態決策熱加載：不停機、零中斷，做到按日自動加載全新 Frontier 策略。
